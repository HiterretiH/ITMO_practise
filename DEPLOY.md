# Инструкция по развертыванию

Данная инструкция описывает процесс развертывания сервиса проверки оформления ВКР с использованием Docker. Приложение состоит из двух контейнеров: бэкенд (Spring Boot) и фронтенд (Angular).


## Системные требования

- Операционная система: Windows, macOS, Linux
- Установленный Docker (версия 20.10 или выше)
- Установленный Git (для клонирования репозитория)
- Свободные порты: 7727 (бэкенд), 7272 (фронтенд)

## Установка Docker

Если Docker не установлен, скачайте и установите его с официального сайта:

- **Windows / macOS**: https://www.docker.com/products/docker-desktop/
- **Linux**: следуйте инструкции для вашего дистрибутива (обычно `sudo apt install docker.io docker-compose`)

После установки убедитесь, что Docker запущен:
```bash
docker --version
docker-compose --version
```

## Клонирование репозитория

Откройте терминал (командную строку) и выполните команду:

```bash
git clone https://github.com/HiterretiH/ITMO_practise.git
cd ITMO_practise
```

## Настройка портов (при необходимости)

По умолчанию приложение использует следующие порты:

| Сервис | Порт | Назначение |
|--------|------|------------|
| Бэкенд | 7727 | API для проверки документов |
| Фронтенд | 7272 | Веб-интерфейс пользователя |

После запуска приложение будет доступно по адресу: **http://localhost:7272**

Если эти порты заняты на вашей машине, отредактируйте файл `docker-compose.yaml`:

```yaml
services:
  backend:
    ports:
      - "ЖЕЛАЕМЫЙ_ПОРТ:7727"   # левая часть — внешний порт, правую не трогать

  frontend:
    ports:
      - "ЖЕЛАЕМЫЙ_ПОРТ:7272"   # левая часть — внешний порт, правую не трогать
```

После изменения портов фронтенд будет доступен по новому адресу: `http://localhost:НОВЫЙ_ПОРТ`

## Запуск приложения

В директории с проектом (там, где лежит файл `docker-compose.yaml`) выполните:

```bash
docker-compose up -d
```

Параметр `-d` означает "detached mode" — контейнеры будут работать в фоновом режиме.

### Что происходит при запуске

1. Docker читает файл `docker-compose.yaml`
2. Собирает образ для бэкенда из `./backend/Dockerfile` (Java + Spring Boot)
3. Собирает образ для фронтенда из `./frontend/Dockerfile` (Node.js + Angular)
4. Запускает оба контейнера
5. Фронтенд ожидает запуска бэкенда (благодаря `depends_on`)

### Полная пересборка без использования кеша

Если вы обновили код и хотите пересобрать образы с нуля (без использования кеша Docker), выполните:

```bash
docker-compose build --no-cache
docker-compose up -d
```

Это может быть полезно в следующих ситуациях:
- Изменились зависимости (`package.json` или `build.gradle`)
- Возникли странные ошибки, которые могут быть связаны с закешированными слоями
- Вы хотите гарантированно получить чистую сборку

Обратите внимание: `--no-cache` замедляет сборку, так как Docker не использует сохранённые промежуточные слои. В обычных ситуациях достаточно `docker-compose up -d --build`.

### Логи запуска

Чтобы увидеть процесс сборки и запуска в реальном времени (без фонового режима):

```bash
docker-compose up
```

Чтобы посмотреть логи уже запущенных контейнеров:

```bash
docker-compose logs
docker-compose logs backend   # только бэкенд
docker-compose logs frontend  # только фронтенд
```

## Проверка работоспособности

### Проверка контейнеров

Убедитесь, что оба контейнера запущены и работают:

```bash
docker ps
```

Вы должны увидеть два контейнера: `practise_backend` и `practise_frontend` со статусом `Up`.

### Проверка бэкенда

Откройте в браузере или выполните команду:

```bash
curl http://localhost:7727/v1/validate/jobs/test
```

(Сервер вернёт ошибку 404 — это нормально, значит сервер запущен и отвечает.)

### Проверка фронтенда

Откройте браузер и перейдите по адресу:

```
http://localhost:7272
```

Вы должны увидеть главную страницу сервиса проверки ВКР.

## Использование приложения

После успешного запуска приложение доступно по адресу **http://localhost:7272**

Порядок работы:
1. Загрузите файл в формате `.doc` или `.docx` (перетаскиванием или через кнопку "Выбрать файл")
2. Нажмите кнопку **«Проверить»**
3. Дождитесь окончания проверки (автоматический переход на страницу результатов)
4. Просмотрите сводку и список ошибок
5. При необходимости скачайте PDF-отчёт кнопкой **«Скачать PDF-отчёт»**
6. Для новой проверки нажмите **«Новая проверка»** на странице результатов

## Остановка приложения

Чтобы остановить контейнеры:

```bash
docker-compose down
```

Чтобы остановить и удалить все данные (включая volumes):

```bash
docker-compose down -v
```

## Пересборка после изменений кода

### Обычная пересборка (с использованием кеша)

Если вы внесли изменения в код и хотите пересобрать контейнеры:

```bash
docker-compose up -d --build
```

Флаг `--build` заставляет Docker пересобрать образы перед запуском. Кеш используется для ускорения сборки.

### Полная пересборка (без кеша)

Если обычная пересборка не помогает или вы изменили зависимости:

```bash
docker-compose build --no-cache
docker-compose up -d
```

## Устранение неполадок

### Конфликт портов

**Ошибка:** `port is already allocated` или `bind: address already in use`

**Решение:** Измените внешние порты в `docker-compose.yaml` (как описано выше) или остановите программу, которая использует нужный порт:

```bash
# Найти процесс, использующий порт (Linux/macOS)
sudo lsof -i :7272
sudo lsof -i :7727

# На Windows:
netstat -ano | findstr :7272
```

### Контейнер не запускается

**Ошибка:** контейнер выходит сразу после запуска

**Решение:** Посмотрите логи:

```bash
docker-compose logs backend
docker-compose logs frontend
```

### Фронтенд не видит бэкенд

Если фронтенд запущен, но при попытке загрузить файл возникает ошибка соединения:

1. Проверьте, что бэкенд запущен: `docker ps | grep backend`
2. Проверьте логи бэкенда: `docker-compose logs backend`
3. Убедитесь, что в файле `frontend/src/config/env.ts` указан правильный адрес бэкенда для Docker-окружения

### Очистка и перезапуск с нуля

Если ничего не помогает, можно полностью очистить окружение и пересобрать заново:

```bash
docker-compose down -v          # остановить и удалить контейнеры с volumes
docker system prune -a          # очистить неиспользуемые образы (осторожно!)
docker-compose up -d --build    # пересобрать и запустить
```
